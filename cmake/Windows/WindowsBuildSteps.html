Thanks to the help of Joe & Liang, I now have a fully functional Windows build. There are still a few pieces I haven’t fully generalized, but using these steps and my clone as a starting point, anyone should be able to get up and running fairly quickly.

 

Some notes before we get started:

 

·         ALEMBIC_ROOT defines where the alembic source is stored (e.g. c:\source\alembic)

 

·         ALEMBIC_OUT defines where the alembic output is built (e.g. c:\output\alembic)

 

·         The scripts determine the system architecture (x86 vs. i64) from the system env var PROCESSOR_ARCHITECTURE. When set to amd64, the scripts will build the 64-bit version into %ALEMBIC_OUT%\i64, otherwise (typically when set to x86) the 32-bit version will be built to %ALEMBIC_OUT%\x86.

 

·         Debug versions can be built by passing “db:” into my scripts. In this case the output will be built to %ALEMBIC_OUT%\i64_db & %ALEMBIC_OUT%\x86_db respectively.

 

·         I don’t currently have the GLew libraries installed, so locally I have commented out the line “ADD_SUBDIRECTORY( GLUtil )” in %ALEMBIC_ROOT%\lib\Alembic\CMakeLists.txt.

 

The steps:

 

1)      Sync with my clone: https://pete-win.googlecode.com/hg/.  This will only be necessary until Joe merges my clone into the trunk.

 

2)      Install cmake from http://www.cmake.org/; add the cmake\bin to your path

 

3)      Install python from http://www.python.org/download/, and add the python dir to you path

 

4)      Download the following and place the source in the corresponding folders in %ALEMBIC_ROOT%\contrib:

 

Destination                         Download location                                         

 

boost_1_42_0\                 http://www.boost.org/users/history/version_1_42_0

hdf5-1.8.5-patch1\          http://www.hdfgroup.org/ftp/HDF5/prev-releases/hdf5-1.8.5-patch1/src/

ilmbase-1.0.2\                   http://www.openexr.com/downloads.html

openexr-1.6.1\                 http://www.openexr.com/downloads.html

 

5)      If not already present, download ZLIB 1.2.5 from http://zlib.net/ and build a static library. Edit the “ZLIB_ARGS” line of init_Alembic.cmd to point to the ZLIB include directory and the static .LIB file (the actual filename, not just the folder).

 

6)      Edit the “srcDir” line of build_ilmbase.cmd to point to the folder containg the VC solution file for your environment (vc7, vc8, vc9).

 

7)      Set up static-library version of the VC projects for IlmBase. You have two options: (1) extract the files from my IlmBase-vc9.zip into contrib\ilmbase-1.0.2\vc\vc9, or (2) manually perform the following steps to convert the projects (answering “Yes” anytime you are asked to save changes):

 

·         Open the IlmBase.sln from contrib\ilmbase-1.0.2\vc\vc8\IlmBase or vc7\Ilmbase.

 

·         Select Build > Configuration Manager and for all 4 combinations of x86/Win32 & Release/Debug, *un*check the Build column for the createDLL project.

 

·         Open the Solution Explorer, and control-select Half, Iex, IlmThread and Imath. Right-click and select Properties. Change “Configuration:” to “All Configurations” and “Platform:” to “All Platforms”. Under “Configuration Properties”, change “Configuration Type” to “Static Library (.lib)”.

 

·         Under “Build Events”, select “Post-Build Event” and change “Excluded From Build” to “Yes”.

 

·         In the Solution Explorer, shift-select all the project. Right-click and select Properties (not necessary if you still have the Properties dialog open). With “Platform:” still set to “All Platforms”, change “Configuration:” to “Release” and under “C/C++” > “Code Generation”, change “Runtime Library” to “Multi-threaded (/MT)”.

 

·         Change “Configuration:” to “Debug” and change “Runtime Library” to “Multi-threaded Debug (/MTd)”.

 

·         For each Project / Configuration / Platform, remove “OPENEXR_DLL” from the “Preprocessor Definition” under “C/C++” > “Preprocessor”. Alternatively you can do a global search/replace on all the VCPROJ files to change “OPENEXR_DLL;” and “;OPENEXR_DLL” to “”.

 

1)      Now you can either run init_all.cmd (if you feel confident J) or the individual setup scripts (build_boost.cmd, build_hdf.cmd, build_ilmbase.cmd and init_Alembic.cmd).

 

2)      Now that every is set up correctly, you can simply run build_Albemic.cmd, any time you update the alembic source.

 

It’s possible that I may have left something out, or that these steps may need some tweaking for your build environment. Hopefully this will get you off to a good start.

 

Good luck!
